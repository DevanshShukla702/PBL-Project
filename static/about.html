How CGEE Works

Explore the internal technical architectures that drive our Contextual Graph ETA estimations globally inside the Bengaluru data zone.

Step 1: Graph Construction

We generate a deeply annotated multi-directed geographic graph structure mapping OSM boundary layouts seamlessly onto Python network topology.

import osmnx as ox
G = ox.graph_from_place("Bengaluru")

Step 2: Shortest Path Routing

Dijkstra's shortest-path runs implicitly using custom physical length extraction preventing multi-hop path distortions, alongside an edge-penalty solver computing k-distinct alternates.

path = nx.shortest_path(G, o, d, weight=dyn_w)

Step 3: Feature Engineering

The router strips the pure paths strictly back out into isolated discrete edge frames mapping metadata context arrays independently (road_type, length_km, severity_prob).

X = [[hour, len_km, is_residential, ...]]

Step 4: ML Prediction

Multi-horizon XGBoost machines execute independently in memory mapping physical path segments directly into timeline specific speeds (1hr models differ statistically from 4hr models).

speed = model_1hr.predict(X_scaled)

Step 5: ETA Aggregation

Predicted speeds mapped against absolute distances summarize to compute global path ETAs gracefully scaled down strictly applying synthetic contextual friction delays.

ETA = Σ (distance_i / speed_i) × 60
