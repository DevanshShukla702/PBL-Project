How CGEE™ Works

A patented system combining graph routing with machine learning for context-aware travel time prediction.

1
Graph Construction
The road network of Bengaluru is loaded from OpenStreetMap using OSMnx. Intersections become nodes, road segments become weighted edges with real distances.
2
Shortest Path Routing
Given source and destination coordinates, the nearest graph nodes are found and the shortest path is computed using Dijkstra's algorithm weighted by road segment length.
3
Feature Engineering (Per Segment)
For each road segment on the route, a 7-feature vector is constructed: lag speed, hour of day (sine/cosine encoded), peak hour flag, incident flag, and incident severity.
4
Multi-Horizon ML Prediction
Three independently trained XGBoost models predict traffic speed for 1-hour, 2-hour, and 4-hour future windows. Each model captures different congestion patterns across time.
5
ETA Aggregation
Segment ETAs are computed as Distance ÷ Predicted Speed, then summed across all route segments. Uncertainty bands and confidence scores are derived from prediction variance.
Patent Information

The Contextual Graph ETA Engine (CGEE™) is a published invention by Devansh Shukla and Kirti Paliwal, Department of Computer Science & Engineering, Manipal University Jaipur, Rajasthan, India.

  • Segment-level multi-horizon predictive modeling integrated with graph routing
  • Dynamic contextual scaling function applied post-prediction
  • Modular architecture enabling independent model versioning
  • ETA computation on edges rather than route-average
