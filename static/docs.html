API Reference

CGEE™ exposes a simple REST API for real-time ETA prediction.

http://localhost:8000
Replace with your deployed domain in production.
Data Constraints
Coordinates must be within Bengaluru bounds:
Latitude: 12.70 to 13.20  ·  Longitude: 77.30 to 77.80
GET /health

Check engine initialization status. Use this to verify the road graph and models are fully loaded into memory.

Response Example
{
  "status": "ok",
  "engine_ready": true,
  "graph": {"nodes": 34521, "edges": 82104},
  "models": ["1_hour", "2_hour", "4_hour"],
  "route_cache": {"hits": 12, "misses": 5, "maxsize": 256}
}
POST /predict-route-eta

Predict multi-horizon ETA for a route. Calculates optimal paths via graph routing and evaluates segment-level ETA using contextual ML models.

Request Body
{
  "source": {"lat": 13.1007, "lon": 77.5963},
  "destination": {"lat": 12.8399, "lon": 77.6770}
}
Successful Response (200 OK)
{
  "routes": [
    {
      "route_id": 1,
      "label": "Fastest Route",
      "color": "#3B82F6",
      "eta_minutes": {
        "1_hour": {"estimate": 38.2, "lower_bound": 34.1, "upper_bound": 43.7},
        "2_hour": {"estimate": 44.6, "lower_bound": 39.2, "upper_bound": 51.3},
        "4_hour": {"estimate": 52.1, "lower_bound": 44.8, "upper_bound": 60.9}
      },
      "confidence": {"1_hour": 0.87, "2_hour": 0.81, "4_hour": 0.73},
      "meta": {
        "distance_km": 28.4,
        "segments": 112,
        "incident": true,
        "incident_segments": 3,
        "avg_incident_severity": 0.41,
        "route_geometry": [{"lat":13.1,"lon":77.5}, "..."]
      }
    }
  ]
}
Error Responses:
• 422 Unprocessable Entity — Invalid coordinates (out of bounds).
• 503 Service Unavailable — Engine not ready yet.
• 500 Internal Server Error — Pathfinding or computation failed.
GET /cache/clear

Clear the route cache. Useful after graph updates or memory cleanup.
