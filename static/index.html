Core Capabilities

Built for real-world accuracy

Each prediction is the result of graph routing, probabilistic incident modelling, and ML-driven speed forecasting across three time horizons.

🗺️
Live Road Graph
Built from OSMnx with 154K nodes and 392K edges. Routes are computed over the real Bengaluru network — not approximate straight-line distances.
⚡
Multi-Horizon Prediction
Three independently trained XGBoost models produce ETA estimates at 1-hour, 2-hour, and 4-hour lookahead windows with confidence bands.
🛣️
Three Route Types
Dijkstra + iterative edge-penalty generates the Fastest, Alternate, and Scenic routes for each query — all with independent ETA calculations.
⚠️
Contextual Incidents
A per-segment Beta(2,5) incident model with spatial propagation adjusts speed predictions — making the engine respond to live friction.
📊
Departure Intelligence
Smart departure chips compare current vs future ETAs using the correct model for each horizon, helping you choose the optimal departure time.
🔌
REST API
Every capability is accessible via a clean FastAPI-served JSON endpoint. Integrate CGEE's ML pipeline directly into your own applications.
Engine Architecture

Prediction pipeline overview

From a coordinate pair to a multi-route ETA response in under 3 seconds — here's what the engine does behind the scenes.

Graph snap — nearest OSM node found via k-d tree lookup
Dijkstra × 3 — edge-penalty method produces three distinct routes
Incident simulation — Beta-distributed, road-class-aware per segment
XGBoost inference — per-segment speed prediction across three horizons
Contextual modulation — patent-filed adjustment for incident severity
